# Feedback Channels

## What feedback is wanted

We actively want feedback about:

- item ambiguity
- flawed canonical solutions
- taxonomy gaps
- subject imbalance
- leakage risk
- licensing or provenance concerns
- reviewer-rubric clarity

## Suggested channels

- GitHub issues for concrete benchmark problems
- GitHub discussions for broader scope or governance suggestions
- direct editorial contact for sensitive leakage or conflict issues

## Feedback loop

1. Public feedback is logged.
2. Editorial triage assigns a route: metadata fix, item review, taxonomy review, or governance review.
3. Outcomes appear in release notes, correction notes, or manuscript updates as appropriate.

This project should improve in public, not behind the scenes.
