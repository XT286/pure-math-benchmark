# Project Charter

## Mission

Build a public benchmark standard for evaluating `research-style pure mathematics` in language models, with enough mathematical, editorial, and governance rigor that it can be taken seriously by the broader mathematical community.

## Core commitments

The project commits to:

- broad pure-math representation rather than narrow benchmark convenience
- explicit `MSC2020` alignment
- expert review and correction processes
- transparent scope and anti-goals
- versioned public releases
- long-term community participation

## Benchmark philosophy

The benchmark should measure how well systems handle mathematical reasoning that resembles real mathematical work:

- recognizing and using the right definitions
- checking hypotheses carefully
- selecting the right theorems or methods
- building valid proof skeletons
- producing or rejecting examples and counterexamples
- making genuine partial progress without bluffing

## Public-standard ambition

To become a standard, the repository must be more than a dataset:

- the item schema must be auditable
- the review process must be legible
- the benchmark niche must be clearly explained
- governance must exist from the start
- release and correction policies must be stable

## Success conditions

The charter is fulfilled when the project becomes something that mathematicians can:

- critique seriously
- contribute to responsibly
- cite in evaluation discussions
- use without guessing how items were selected or judged
